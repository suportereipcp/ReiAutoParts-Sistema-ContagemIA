Controle de Acesso

Modelo de Permissões

O sistema usa granularidade híbrida: a página agrupa visualmente; a atividade é a unidade controlada.

efetivo(usuário) = (união das atividades dos grupos ∪ concessões) − revogações

Revogação sempre vence sobre concessão.

Modelo de Dados

acesso_grupos

  • id — UUID PK
  • nome — UNIQUE NOT NULL
  • descricao — opcional
  • criado_em / atualizado_em

acesso_grupo_atividades

  • grupo_id — FK → acesso_grupos
  • atividade_id — ref. catálogo
  • PK composta (grupo_id, atividade_id)

acesso_usuarios

  • id — UUID do Supabase auth
  • email, nome
  • sincronizado_em — cache read-only

acesso_usuario_overrides

  • usuario_id — FK
  • atividade_id — ref. catálogo
  • efeito — 'conceder' | 'revogar'
