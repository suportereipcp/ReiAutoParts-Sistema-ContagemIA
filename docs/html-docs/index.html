Visão Geral

Sistema de contagem automatizada de peças para a Rei AutoParts. Roda em Edge PC Windows, conectado a câmeras industriais Keyence IV4 via TCP. Arquitetura edge-first: operações críticas nunca dependem de rede.

Node.js 20

Runtime ESM

SQLite

Banco local

Supabase

Sync remoto

Keyence IV4

Câmeras TCP

Arquitetura

Fluxo de Dados

Câmera IV4 (TCP 8500) → TCP Listener → Camera Manager → Domain
Domain → SQLite (contagem + outbox)
Domain → WebSocket Hub → Monitor 1 / Monitor 2 (TV kiosk)
Sync Worker → Supabase (schema: sistema_contagem) [assíncrono]
Supabase → Reverse Poller (embarques/OPs/operadores a cada 30s)

Invariantes Críticas

  • • 1 sessão ativa por câmera — índice parcial único
  • • Comando antes de escuta — câmera só emite após OE,1
  • • Idempotência no sync — UNIQUE constraints previnem duplicatas
  • • Leitura sempre local — abertura de sessão nunca consulta Supabase

Sync & Supabase

Sincronização assíncrona bidirecional entre SQLite local e Supabase (schema sistema_contagem).

Edge → Supabase (Outbox)

  • sessoes_contagem
  • eventos_log
  • etiquetas_caixa / partes
  • acesso_grupos / atividades / overrides
  • embarques_status

Supabase → Edge (Reverse Poller)

  • embarques
  • ordens_producao
  • operadores
  • acesso_usuarios (auth cache)

Câmera Keyence IV4

2 câmeras Keyence IV4-600CA conectadas via TCP na porta 8500. Protocolo texto com comandos e respostas.

Comandos Principais

Operação

Iniciar o sistema

npm run dev          # desenvolvimento
npm start            # produção

Variáveis obrigatórias (.env)

  • NEXT_PUBLIC_SUPABASE_URL
  • SUPABASE_SERVICE_ROLE_KEY
  • CAMERA_1_IP / CAMERA_2_IP

Banco de dados

SQLite em ./data/contagem.db. Migrations aplicadas automaticamente no boot.
